NHK ラジオ語学を、
毎日 自動 DL して Podcast で聴く。

聴き逃し配信は約1週間で消える。
Mac (または Linux) に常駐させて、毎朝勝手にエピソードを蓄積。
iPhone の Apple Podcasts で購読すれば、通勤中もオフライン再生できる。

$
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/asari-mtr/nhk_gogaku_2026/main/bootstrap.sh)"
macOS / Linux / WSL 対応 · MIT-ish ライセンス · 個人利用前提

特徴

⏰

毎日自動DL

launchd / cron で朝9時に新エピソードを取得。スリープから復帰後も追いかけ実行。

🎙

無劣化保存

配信そのままの AAC を .m4a で保存。5分番組で約1.7MB。mp3変換も可能。

📱

iPhone で Podcast

RSSフィードを自動生成。Apple Podcasts で「URLから追加」して外でオフライン再生。

🌐

Web UI

ブラウザで番組選択・DL実行・QRコード表示。ターミナル不要で全機能操作可能。

🎶

Apple Music 対応

メタデータ + 番組ロゴ埋め込み済み。ライブラリに追加するだけでアルバム化。

🔁

再放送スキップ

半期で同じ番組が繰り返されても重複DLしない。短いDLは自動リトライ。

インストール

ターミナルに1行コピペするだけ。
不足する依存があれば、それを画面に表示して案内します（勝手にインストールはしません）。

$
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/asari-mtr/nhk_gogaku_2026/main/bootstrap.sh)"

必要な外部ツール

ツール用途必須 / 任意
gitGitHub から取得 / 更新必須
curlbootstrap.sh / API アクセス必須
python3 (3.10+)スクリプト実行 (標準ライブラリのみ)必須

OS別 インストールコマンド

macOS
brew install git curl python ffmpeg uv
Ubuntu / Debian
sudo apt install git curl python3 ffmpeg
curl -LsSf https://astral.sh/uv/install.sh | sh
Fedora / RHEL
sudo dnf install git curl python3 ffmpeg
curl -LsSf https://astral.sh/uv/install.sh | sh
Arch / Manjaro
sudo pacman -S git curl python ffmpeg uv
Windows ネイティブ環境では動きません。 PowerShell（管理者）で wsl --install -d Ubuntu を実行して WSL を入れてから、その中で上記コマンドを実行してください。

FAQ

これは NHK 公式のツールですか？

いいえ、有志による 非公式 ツールです。生成される Podcast フィードのタイトルにも [非公式] を付け、作者は Personal Archive として明示しています。NHK の聴き逃し配信を私的利用の範囲でローカル保存することを目的としています。

外でも聴けますか？

はい。iPhone の Apple Podcasts に購読 → 家のWi-Fi圏内にいる間に自動DL → 外出先でオフライン再生できます。同様に Pocket Casts や Overcast でも可能です。

どの番組が取れますか？

NHK ラジオの語学番組ほぼすべて。英語・中国語・ハングル・フランス語・ドイツ語・イタリア語・スペイン語・ポルトガル語・ロシア語・タイ語 (10言語、18番組) に対応。Web UI 上のチェックボックスで選択するだけです。

音質は？

NHK の配信そのまま (AAC 48kbps) を 無劣化 で保存します。語学番組に十分な音質で、5分番組で約1.7MB。mp3変換も可能 (64/128/192 kbps から選択)。

常に Mac を起動しておく必要がありますか？

launchd を使えば スリープ中はジョブが待機 → 起床後すぐ追いかけ実行 されるので、必要なときだけMacを開けば十分です。完全シャットダウンでなければOK。

もっと簡単な配布形式 (バイナリ) はありますか？

現状はソーススクリプト形式のみ。ffmpeg は別途インストールが必要なため、Python だけバイナリ化しても根本的な依存は減らせません。macOS/Linux はだいたい Python 3 が標準で入っているので、現状の形が最もシンプルです。

アンインストールは？

./scripts/uninstall.sh で launchd / cron から解除されます。DL済みファイル (downloads/) は明示的に削除しない限り残ります。
